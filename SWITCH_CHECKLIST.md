# Чеклист перехода на новую админку и кабинет мерчанта

## Перед деплоем

- [ ] Запустить миграции: `php artisan migrate`
- [ ] Запустить сидер прав: `php artisan db:seed --class=AdminPermissionSeeder`
- [ ] Убедиться что `.env` содержит:
  - `SANCTUM_STATEFUL_DOMAINS` включает домены админки и кабинета
  - CORS настроен для новых доменов
- [ ] Собрать frontend: `npm run build -w admin-panel && npm run build -w merchant-panel`
- [ ] Проверить что очередь `export` запущена (для CSV-экспорта)

## Безопасность

- [ ] Admin API защищён guard `auth:admin` + `can:*` permissions
- [ ] Cabinet API защищён guard `auth:cabinet` + `EnsureCabinetMerchant`
- [ ] Финансовые операции (withdrawal, convert, insurance update) — только merchant admin
- [ ] Impersonate-токен с TTL 1 час
- [ ] Нельзя отвязать последнего admin мерчанта

## Rollback план

2. API v2 роуты остаются доступными, но frontend переключается
3. Данные обратно совместимы (нет деструктивных миграций)
