# 📖 Глоссарий терминов PSP Platform

> Словарь терминов для понимания платёжной системы

---

## A

### Amount
Сумма платежа в валюте. Может отличаться от `init_amount` из-за уникализации.

### Amount Range
Диапазон для уникализации сумм. Если `amount_range = 10`, то к сумме 5000 может быть добавлено случайное число от 1 до 10, чтобы различить платежи.

### API Token
Bearer токен для авторизации API запросов. Выдаётся мерчанту при регистрации.

### Апелляция (Appeal)
Процесс оспаривания статуса платежа. Плательщик загружает чек, трейдер проверяет.

---

## B

### Balance
Баланс мерчанта. Может быть в USDT (по умолчанию) или в фиате (при неттинге).

### Bank Detection
Сервис определения банка по номеру карты (BIN).

### BIN (Bank Identification Number)
Первые 6 цифр номера карты, идентифицирующие банк-эмитент.

---

## C

### Callback
HTTP уведомление о смене статуса платежа, отправляемое на `callbackUri` мерчанта.

### Callback URI
URL, на который отправляются уведомления о статусе платежа.

### Cascade / Каскад
Механизм автоматического выбора провайдера. При отказе одного провайдера система переключается на следующий.

### Cascade Attempt
Одна попытка создания платежа через конкретного провайдера в рамках каскада.

### Cascade Percent
Вес провайдера в каскаде (0-100). Чем выше, тем чаще выбирается.

### C2C (Card to Card)
Метод оплаты — P2P перевод с карты на карту.

### Commission
Комиссия за платёж в процентах.

### Credentials (Креды)
Набор данных для доступа: merchantId, token, secretKey.

---

## D

### Deal
Заявка в системе Trade. Аналог Payment, но на стороне P2P процессинга.

### Device
Устройство трейдера (смартфон) с приложением SMS Forwarder.

### Dispute / Диспут
Спор по платежу. Создаётся при несогласии с результатом.

---

## E

### ECOM
E-commerce эквайринг. Классическая оплата картой с 3D Secure.

### Expired
Статус платежа — истёк срок ожидания оплаты (15 минут).

### External ID
ID платежа в системе провайдера (transaction_id).

---

## F

### Failed
Статус платежа — не удалось создать (нет реквизитов, ошибка провайдера).

### Finished
Статус платежа — успешно завершён.

### Flow
React-приложение — платёжная страница для плательщиков.

### FTD (First Time Deposit)
Первый депозит пользователя. Тип payer.

---

## G

### Gateway
PHP-класс провайдера в Aggregator, реализующий интерфейс Gatewable.

### Gatewable
Интерфейс для провайдеров: createInvoice, processCallback, getStatus, etc.

### Gostscope
Единый дашборд для мониторинга всех White-label проектов.

---

## H

### H2H (Host-to-Host)
Тип интеграции — прямое API взаимодействие без редиректов.

### HMAC-SHA256
Алгоритм подписи запросов. Используется для Signature header.

### HUMO
Платёжная система Узбекистана.

---

## I

### Init Amount
Изначальная сумма платежа (до уникализации).

### Invoice / Инвойс
Реквизиты для оплаты: номер карты/телефона, ФИО, банк, комментарий.

### IP Whitelist
Список разрешённых IP для доступа к API.

---

## K

### KGS
Киргизский сом (валюта).

### KZT
Казахстанский тенге (валюта).

---

## L

### Limit
Лимит на реквизите (карте) трейдера — максимальная сумма в день.

### Limit One
Лимит на одну операцию у трейдера.

---

## M

### Merchant / Мерчант
Клиент платформы (интернет-магазин, сервис). Имеет merchantId, token, secretKey.

### Merchant ID
Уникальный идентификатор мерчанта для API.

### Method / Метод
Способ оплаты: sbp, c2c, nspk, ecom, etc.

### Method Type
Группа методов: payin, payout, bank-to-bank.

---

## N

### Netting / Неттинг
Режим без конверсии. Баланс ведётся в фиате, а не в USDT. При неттинге `rate = null`.

### NSPK
Национальная система платёжных карт. QR-оплата через СБП.

---

## O

### OpenSearch
Поисковый движок для логов. Аналог Elasticsearch.

### Order ID
Уникальный ID заказа от мерчанта. Используется для идентификации платежа.

---

## P

### Payer
Плательщик. Объект с данными: userId, userIp, customerName, phone, etc.

### Payin
Приём платежа от плательщика на счёт мерчанта.

### Payment
Основная сущность — платёж (payin или payout).

### Payment Page
Тип интеграции — редирект на платёжную страницу (Flow).

### Payout
Выплата средств получателю со счёта мерчанта.

### Pending
Статус платежа — ожидает оплаты, реквизиты выданы.

### P2P (Peer-to-Peer)
Перевод между физическими лицами.

### Provider / Провайдер
Внешний или внутренний сервис для обработки платежей.

### Provider Group
Группа провайдеров с общими лимитами.

---

## Q

### Queue
Очередь задач (Redis). Используется для async операций.

---

## R

### Rate / Курс
Курс обмена фиат → USDT. Источник: Binance, Bybit, etc.

### Rate Service
Микросервис агрегации курсов с криптобирж.

### Recipient
Получатель — ФИО или данные для payout.

### Requisite / Реквизит
Банковская карта или счёт трейдера для приёма/отправки денег.

### RUB
Российский рубль (валюта).

---

## S

### SBP / СБП
Система быстрых платежей (Россия). Переводы по номеру телефона.

### Secret Key
Секретный ключ мерчанта для HMAC подписи.

### Sender Bank
Банк отправителя. Указывается в payer.senderBank.

### Settlement Period
Период расчёта. T+0 (сразу) или T+1 (следующий рабочий день).

### Signature
HMAC-SHA256 подпись запроса. Header: `Signature: {hash}`.

### SLA (Service Level Agreement)
Соглашение об уровне сервиса. Время ответа ≤ 15 минут.

### SMS Parser
Компонент Trade для автоматического парсинга SMS от банков.

### State
Статус платежа: created, pending, in_check, dispute, finished, expired, failed, canceled.

- **created** — платёж создан, но инвойс (реквизиты) ещё не выдан. Промежуточный статус.
- **pending** — реквизиты выданы, ожидает оплаты.
- **in_check** — на проверке.
- **dispute** — создан диспут/апелляция.
- **finished** — успешно завершён.
- **expired** — истёк срок ожидания оплаты.
- **failed** — ошибка при создании.
- **canceled** — отменён.

### STD (Standard)
Стандартный пользователь. Тип payer.

### Support Service
Микросервис тикетов и диспутов с Telegram интеграцией.

---

## T

### Telescope
Laravel инструмент для отладки. Показывает requests, exceptions, jobs, queries.

### Timeout
Максимальное время ожидания в каскаде (секунды).

### Token
Bearer токен для API авторизации.

### Trade
Внутренний P2P сервис. Управляет трейдерами и реквизитами.

### Trader / Трейдер
Физическое лицо с банковскими картами для P2P операций.

### Transaction ID
ID транзакции у провайдера. Хранится в invoice.transaction_id.

### Trusted
Доверенный пользователь с историей успешных платежей. Тип payer.

---

## U

### User ID
ID пользователя в системе мерчанта. Передаётся в payer.userId.

### User IP
IP адрес плательщика. Используется для антифрода.

### USDT
Стейблкоин, привязанный к доллару. Основная валюта баланса.

### UZS
Узбекский сум (валюта).

### UZCARD
Платёжная система Узбекистана.

---

## V

### Vertical
Категория бизнеса мерчанта: gambling, forex, crypto, etc.

---

## W

### Webhook
HTTP уведомление между сервисами.

### White-label (ВЛ)
Брендированная копия платформы для отдельного клиента.

### Whitelist
Список разрешённых IP или карт.

## Числовые сокращения

| Сокращение | Значение |
|------------|----------|
| T+0 | Расчёт в тот же день |
| T+1 | Расчёт на следующий рабочий день |
| 3DS | 3D Secure — протокол защиты карточных платежей |
| 2FA | Двухфакторная аутентификация |
| 24/7 | Круглосуточно, 7 дней в неделю |

## Коды ошибок

| Код | Название | Описание |
|-----|----------|----------|
| 401 | Unauthorized | Неверный токен или подпись |
| 403 | Access Denied | IP не в whitelist |
| 40001 | Invalid orderId | Некорректный или дублирующийся orderId |
| 40002 | Requisites Busy | Нет свободных реквизитов |
| 40003 | Invalid Method | Метод не подключён |
| 40004 | Insufficient Funds | Недостаточно средств |
| 40301 | User Blocked | Пользователь заблокирован |
| 40302 | Anti-spam | Слишком много запросов |
| 40401 | Not Found | Мерчант/платёж не найден |
| 40403 | No Methods | Методы не подключены |
| 40901 | Duplicate | Дубликат платежа |
