# Кабинет мерчанта — документация

## Обзор

Кабинет мерчанта — отдельное SPA-приложение для управления платежами, выплатами, балансами и настройками мерчанта. Доступно по отдельному URL (например, `merchant.example.com`).

## Создание мерчанта

1. Войдите в админ-панель
2. Перейдите в раздел "Мерчанты" → "Создать мерчанта"
3. Заполните мультишаговую форму:
   - **Шаг 1:** Имя, код, email админа, whitelist IP, неттинг
   - **Шаг 2:** Callback/Success/Fail URI, Telegram
   - **Шаг 3:** Таймауты, ref_share_mode, вертикали
4. После создания отобразятся:
   - Secret Key
   - API Token
   - Логин и пароль пользователей (admin + manager)

**Сохраните эти данные — пароли не показываются повторно.**

## Вход в кабинет

Мерчант входит по email и паролю, которые были созданы при регистрации. Если включена 2FA — потребуется код из Google Authenticator.

### Вход за мерчанта (для администраторов)

1. В админ-панели откройте страницу мерчанта
2. Нажмите кнопку "Войти как мерчант"
3. Откроется кабинет мерчанта в новой вкладке
4. В верхней части будет баннер "Вы вошли как администратор"
5. Все действия записываются в аудит-лог

## Экраны кабинета

### Платежи (главная)
- Таблица всех платежей с группировкой по дням
- Фильтры: ID, сумма, валюта, тип (Pay In/Pay Out/Withdrawal), статус, дата
- Кнопки: "Создать платёж", "Создать выплату" (только для роли Admin)
- Клик по строке — модалка с деталями: сумма, статус, USDT, комиссия, реквизиты
- Действия в модалке: загрузить чек, отправить на проверку, повторить callback

### Операции с балансом
- Вывод средств (USDT → TRC20 кошелёк, требуется 2FA)
- Конвертация (фиат → USDT или USDT → фиат, только при включённом неттинге)
- История операций: выводы, конвертации, пополнения

### Отчёты
- Список CSV-отчётов с параметрами (тип, статус, валюта, период)
- Скачивание готовых отчётов
- Индикатор "В обработке" для генерируемых отчётов

### Уведомления
- Лента уведомлений с группировкой по дням
- Поиск по тексту

### Настройки
4 секции:
1. **Персонализация** — имя пользователя, язык
2. **Настройки мерчанта** — код (readonly), ключ сигнатуры (readonly), Callback/Success/Fail URI, Flow URL, Logo URL, диапазон уникализации суммы
3. **Интеграции** — Telegram бот (readonly), ID чатов
4. **Автовывод** — адрес кошелька, минимальная сумма, процент вывода (требуется 2FA)

## Роли пользователей мерчанта

| Возможность | Admin | Manager |
|---|---|---|
| Просмотр платежей | Все типы | Только payin и payout |
| Создание платежей/выплат | Да | Нет |
| Просмотр баланса | Да | Нет |
| Вывод средств | Да (2FA) | Нет |
| Конвертация | Да | Нет |
| Настройки мерчанта | Да | Нет |
| Настройки автовывода | Да (2FA) | Нет |
| Настройки профиля | Да | Да |

## Описания полей

### Настройки мерчанта

| Поле | Описание |
|---|---|
| Код магазина | Уникальный код мерчанта для идентификации. Не редактируется. |
| Ключ сигнатуры | Секретный ключ для подписи API-запросов. Не редактируется. |
| Callback URI | URL для получения уведомлений о статусе платежей. Используется, если не указан в запросе. |
| Success URI | URL для перенаправления после успешной оплаты. |
| Fail URI | URL для перенаправления после неудачной оплаты. |
| Flow URL | Уникальный адрес платёжной страницы мерчанта. |
| Ссылка на лого | Прямая ссылка на файл с изображением (SVG, PNG). |
| Диапазон уникализации суммы | Например, "1" — сумма меняется на ±1. |
| Токен Telegram-бота | Токен бота, присылающего уведомления об успешных платежах. |
| ID пользователей Telegram | Telegram ID пользователей и чатов для уведомлений. Каждый ID с новой строки. |
| Сумма USDT для автовывода | При достижении указанной суммы на балансе автоматически создаётся вывод средств. |

## White-label

Кабинет поддерживает кастомный брендинг через переменные окружения:
- `VITE_BRAND_NAME` — название бренда (по умолчанию "Goxtex")
- `VITE_BRAND_LOGO` — URL SVG-логотипа (если не указан — отображается только текст)

Для каждого клиента создаётся отдельная сборка с нужными env-переменными (аналогично flow).

## Безопасность

- Все API-эндпоинты защищены Sanctum-авторизацией (guard `cabinet`)
- Middleware `EnsureCabinetMerchant` проверяет привязку пользователя к мерчанту
- Middleware `EnsureMerchantAdmin` ограничивает финансовые операции только для роли Admin
- Операции вывода и настройки автовывода требуют 2FA
- Impersonate-токены одноразовые с TTL 1 час
