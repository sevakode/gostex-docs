## 1. Добавление админа или агента

**Доступ:** admin, superAdmin

### Где найти

Боковое меню -> **Админы и агенты** -> кнопка **Создать**.

![Скриншот: список админов и агентов с кнопкой "Создать"](screenshots/admin-list.png)

### Пошаговая инструкция

1. Перейдите в раздел **Админы и агенты**.
2. Нажмите кнопку **Создать** в правом верхнем углу.
3. Заполните поля:

| Поле | Описание |
|------|----------|
| **Username** | Уникальное имя для входа в панель |
| **Пароль** | Пароль для авторизации. Используйте сложный пароль с буквами, цифрами и спецсимволами |
| **Email** | Электронная почта для уведомлений и восстановления доступа |
| **USDT Wallet** | Адрес криптовалютного кошелька агента для получения комиссионных выплат |
| **Роли** | Выберите одну или несколько ролей из выпадающего списка (подробнее — в разделе 2) |
| **MFA** | Включение двухфакторной аутентификации (Google Authenticator). Рекомендуется включать |
| **Автовыход** | Автоматическое завершение сессии при длительном бездействии |

4. Нажмите **Сохранить**.

![Скриншот: форма создания нового админа/агента](screenshots/admin-create-form.png)

### Редактирование

Для редактирования существующего админа/агента нажмите на его строку в таблице. На странице редактирования доступны те же поля, а также:
- Сброс 2FA (кнопка **Сбросить 2FA** — только для superAdmin).
- Просмотр привязанных офферов и настройка комиссий.

> **Важно:** Удалить админа/агента можно через кнопку удаления в списке. Действие необратимо.

## 2. Назначение ролей

**Доступ:** superAdmin

### Где найти

Боковое меню -> **Выдача ролей** (видно только суперадминистраторам).

Также роли можно назначить при создании или редактировании админа/агента (поле **Роли**).

### Таблица ролей и прав

| Роль | Описание | Доступные разделы |
|------|----------|-------------------|
| **superAdmin** | Полный доступ ко всем функциям | Все разделы, включая системные настройки и выдачу ролей |
| **admin** | Управление трейдерами и настройками | Трейдеры, Админы и агенты, Офферы, Транзакции, Споры, Реквизиты, Методы, Типы методов, Контрагенты |
| **agent** | Управление своими трейдерами | Трейдеры (только свои), Выручка, Выводы |
| **agentBounded** | Ограниченный агент | Трейдеры (только привязанные), Выручка, Выводы |
| **support** | Просмотр и работа со спорами | Трейдеры, Транзакции, Споры, Устройства, Банки, Курсы валют |

![Скриншот: страница выдачи ролей RBAC](screenshots/rbac-assignment.png)

### Как назначить роль

1. Перейдите в **Выдача ролей**.
2. Найдите пользователя в списке.
3. Выберите нужные роли.
4. Сохраните изменения.

> **Важно:** Один пользователь может иметь несколько ролей. Права суммируются.

## 3. Добавление типа метода

**Доступ:** admin

### Где найти

Боковое меню -> **Типы методов** -> кнопка **Создать**.

### Что такое тип метода

Тип метода определяет формат данных для платежа. Например:
- **Банковская карта** — номер карты, ФИО получателя
- **Телефон (СБП)** — номер телефона
- **QR** — QR-код для сканирования
- **NSPK** — ссылка для оплаты через СБП

Тип метода задаёт, какие поля будет заполнять плательщик при оплате и какие реквизиты нужны от трейдера.

### Пошаговая инструкция

1. Перейдите в **Типы методов**.
2. Нажмите **Создать**.
3. Укажите название типа (например, «Карта», «Телефон», «QR»).
4. Сохраните.

![Скриншот: список типов методов](screenshots/method-types-list.png)

> **Важно:** Тип метода привязывается к методу оплаты (раздел 4). Один тип может использоваться несколькими методами.

## 4. Добавление метода оплаты

**Доступ:** admin

### Где найти

Боковое меню -> **Методы** -> кнопка **Создать**.

### Пошаговая инструкция

1. Перейдите в раздел **Методы**.
2. Нажмите **Создать**.
3. Заполните поля:

| Поле | Описание |
|------|----------|
| **Название** | Уникальное название способа оплаты (например, «Сбербанк», «Тинькофф», «QIWI»). После создания изменить нельзя |
| **Банки** | Выберите один или несколько банков, которые поддерживают этот метод |
| **Тип метода** | Выберите тип из выпадающего списка (см. раздел 3) |
| **Реквизит без номера карты в SMS** | Если включено — номер карты не будет фигурировать в SMS-уведомлениях. Полезно для методов по номеру телефона |
| **Доступен внутрибанк** | Разрешить внутрибанковские переводы для этого метода |

4. Нажмите **Сохранить**.

![Скриншот: форма создания метода оплаты](screenshots/method-create-form.png)

### Быстрое добавление банков по валюте

Над списком банков есть выпадающий список **Валюта**. Выберите валюту и нажмите кнопку **Добавить банки** — все банки с указанной валютой автоматически добавятся в список.

> **Важно:** Название метода задаётся один раз при создании и потом не редактируется.

## 5. Добавление трейдера

**Доступ:** admin, superAdmin (агенты видят только своих трейдеров в режиме просмотра)

### Где найти

Боковое меню -> **Трейдеры** -> кнопка **Создать**.

### Пошаговая инструкция

1. Перейдите в раздел **Трейдеры**.
2. Нажмите **Создать**.
3. Заполните основные поля:

#### Основная информация

| Поле | Описание |
|------|----------|
| **Статус** | Активный / Заблокирован / На проверке |
| **Администратор** | Выберите админа или агента, который будет курировать трейдера. Задаётся один раз при создании |
| **Email** | Электронная почта трейдера |
| **Телефон** | Контактный номер |
| **ФИО** | Полное имя трейдера |
| **Пароль** | Пароль для входа в личный кабинет |

#### Telegram-уведомления

| Поле | Описание |
|------|----------|
| **Telegram чат пейинов** | ID чата для уведомлений о входящих заявках (пополнениях) |
| **Telegram чат выплат** | ID чата для уведомлений о выплатах |
| **Telegram чат споров** | ID чата для уведомлений о спорных ситуациях |
| **Токен бота** | API-токен Telegram-бота, который добавлен в группы трейдера |

#### Лимиты и настройки

| Поле | Описание |
|------|----------|
| **API ключ** | Уникальный ключ для аутентификации устройств (SMS Forwarder) и взаимодействия с API |
| **Мин. сумма** | Минимальная сумма заявки для всех реквизитов трейдера. Внимание: при установке все персональные лимиты реквизитов сбрасываются |
| **Интервал взятия пейаутов (мин)** | Пауза между взятием payout-заявок. 0 = нет ограничения |
| **Страховой депозит** | Сумма в USDT, которая блокируется как гарантия. Доступно = баланс - страховой депозит - замороженные |
| **Лимит споров** | Максимальное количество споров, при превышении — автоблокировка |
| **Лимит выплат** | Максимальное количество одновременных заявок на выплату в работе |
| **Лимит платежей** | Максимальное количество одновременных входящих заявок (не в финальном статусе) |
| **Сессия (часы)** | Время жизни авторизации. По истечении — автоматический выход |

#### Настройки выплат

| Поле | Описание |
|------|----------|
| **Время на выплату (мин)** | Сколько минут даётся на выполнение выплаты. По истечении заявка возвращается в пул |
| **Интервал напоминания (мин)** | Через сколько минут отправлять напоминание о невыполненной выплате |
| **Мин. сумма для пула** | Минимальная сумма заявки, которую трейдер видит в пуле выплат |
| **Макс. сумма для пула** | Максимальная сумма заявки в пуле |
| **Дневной лимит по сумме** | Максимальная сумма выплат за день |

#### Дополнительные опции

| Поле | Описание |
|------|----------|
| **Право отмены апелляций** | Разрешает трейдеру самостоятельно отменять апелляции |
| **Таймаут спора (мин)** | Время, после которого спор по export-заявке решается в пользу клиента |
| **Доверенный трейдер на пейауты** | Трейдер может самостоятельно подтверждать выполнение выплат |
| **Режим метчинга** | Включает автоматическое связывание входящих платежей с выплатами (см. раздел 15) |
| **Доверенные клиенты** | Трейдер получает заявки только от клиентов с историей — новые клиенты к нему не попадают |

4. Нажмите **Сохранить**.

![Скриншот: форма создания трейдера — настройки выплат](screenshots/trader-payout-settings.png)

## 6. Добавление банка

**Доступ:** support и выше

### Где найти

Боковое меню -> **Банки** -> кнопка **Создать**.

### Пошаговая инструкция

1. Перейдите в раздел **Банки**.
2. Нажмите **Создать**.
3. Заполните основные поля:

| Поле | Описание |
|------|----------|
| **Название** | Название банка (например, «Сбербанк»). Задаётся один раз |
| **Слаг** | Короткий идентификатор латиницей (например, «sber»). Используется внутри системы |
| **Валюта** | Трёхбуквенный код (RUB, USD, EUR и т.д.) |
| **Имя отправителя для SMS** | Как банк подписывает SMS (например, «900» для Сбера, «Tinkoff»). Нужно для автоматического определения банка при парсинге SMS |
| **Имя отправителя для Push** | Аналогично для Push-уведомлений |

4. Настройте **поля реквизитов** (блок «Настройки полей»). Для каждого поля укажите:
   - Название (например, phone, account)
   - Тип данных (string, number, email)
   - Плейсхолдер (текст-подсказка)
   - Можно добавлять и удалять поля кнопками **+ Добавить поле** и **✕**

5. Настройте **паттерны парсинга SMS** (блок «Паттерны парсинга SMS»). Типы паттернов:

| Тип паттерна | Назначение |
|--------------|------------|
| **Break** | Отсеивание — если SMS содержит это слово/паттерн, она игнорируется (например, «списание», «покупка») |
| **Amount** | Извлечение суммы из SMS |
| **Card End** | Извлечение последних 4 цифр карты |
| **Date** | Извлечение даты транзакции |

Для каждого паттерна укажите:
- **Паттерн (Regex)** — регулярное выражение (например, `/сумма:\s*(\d+)/ui`)
- **Описание** — что делает этот паттерн
- **Порядок** — приоритет применения (меньше = раньше)
- **Активен** — включён ли паттерн

6. Нажмите **Сохранить все изменения на странице**.

![Скриншот: форма создания банка с полями и паттернами](screenshots/bank-create-form.png)

> **Важно:** Паттерны парсинга критически важны для работы автоматики. Без корректных паттернов SMS от этого банка не будут распознаваться. Для добавления «инверсии» (обязательное слово) добавьте слово «инверсия» в описание паттерна.

## 7. Настройка офферов

**Доступ:** admin (агенты видят в режиме просмотра)

### Где найти

Боковое меню -> **Офферы** -> выбрать нужный оффер.

### Что такое оффер

Оффер — это платёжное направление, которое связывает конкретного мерчанта с методом оплаты. Оффер определяет правила обработки заявок: тайминги, комиссии, источники курсов.

### Поля оффера

| Поле | Описание |
|------|----------|
| **Код мерчанта** | Уникальный идентификатор мерчанта в системе |
| **Имя мерчанта** | Отображаемое название мерчанта |
| **Метод оплаты** | Способ оплаты (Сбербанк, Тинькофф, QR и т.д.) |
| **Мин. процент успеха** | Если конверсия падает ниже этого значения — трафик от оффера блокируется. Оставьте пустым для отключения |
| **Период анализа конверсии (мин)** | За какой период считается конверсия |
| **Мин. заявок для конверсии** | Сколько заявок должно быть создано, чтобы система начала проверять конверсию |
| **Срок действия заявки (мин)** | Сколько минут заявка остаётся активной. По истечении — статус «Просрочен» |
| **Автоотмена выплат (часы)** | Через сколько часов payout-заявки автоматически отменяются |
| **Напоминание об ауте (мин)** | Через сколько минут отправлять напоминание о зависшей выплате в чат споров |
| **Интервал между заявками (мин)** | Минимальное время между заявками на одну сумму и реквизит. **Внимание:** если интервал меньше срока действия заявки — возрастает риск апелляций |
| **Процент разброса для автоматики** | Погрешность суммы для автоподтверждения. Например, при 2% и сумме 1000 подтверждаются суммы от 980 до 1020 |
| **Источник ставок** | Откуда берётся курс валют для пополнений (биржа или ручной) |
| **Источник выплаты ставок** | Откуда берётся курс валют для выплат |
| **Только внутренний метчинг** | Если включено — payout подбирается только из этого же оффера |
| **Вертикаль** | Бизнес-категория оффера (игры, форекс, крипто и т.д.) |

![Скриншот: форма настройки оффера](screenshots/offer-form.png)

### Привязка оффера к трейдеру

Привязка выполняется на странице трейдера (раздел 5):

1. Откройте страницу трейдера.
2. В блоке **Офферы трейдера** найдите нужный оффер.
3. Нажмите переключатель в колонке справа для включения/отключения оффера.
4. Настройте:
   - **Комиссия PayIn** — процент трейдера с входящих платежей
   - **Комиссия PayOut** — процент трейдера с выплат
   - **Комиссия агента PayIn/PayOut** — процент агента
   - **Шанс трафика** — какой процент заявок направляется этому трейдеру
5. Используйте переключатели **PayIn** и **PayOut** для раздельного включения типов операций.

![Скриншот: таблица офферов на странице трейдера](screenshots/trader-offers-table.png)

> **Совет:** Кнопки «Отключить все офферы» / «Включить все офферы» позволяют массово управлять офферами трейдера.

## 8. Динамические ставки (комиссии)

**Доступ:** admin, agent

### Где найти

Страница трейдера -> кнопка **Динамические комиссии** (над таблицей офферов).

### Что это

Динамические ставки позволяют устанавливать разные комиссии для разных диапазонов сумм. Например, для мелких платежей (до 5 000) — комиссия 3%, для крупных (от 50 000) — 1.5%.

### Пошаговая инструкция

1. Откройте страницу трейдера.
2. Нажмите кнопку **Динамические комиссии**.
3. Откроется список существующих тарифов. Нажмите **Создать** для добавления нового.
4. Заполните поля:

| Поле | Описание |
|------|----------|
| **Оффер** | Выберите оффер, для которого создаётся тариф |
| **Минимальная сумма** | Нижняя граница диапазона |
| **Максимальная сумма** | Верхняя граница диапазона |
| **Комиссия Payin (%)** | Комиссия трейдера для входящих платежей в этом диапазоне |
| **Комиссия агента PayIn (%)** | Комиссия агента для входящих платежей |

5. Нажмите **Создать**.

![Скриншот: форма создания динамической ставки](screenshots/commission-tiers-form.png)

> **Важно:** Диапазоны сумм не должны пересекаться. Система проверяет пересечения при сохранении. Если сумма заявки попадает в диапазон динамической ставки — применяется она, а не базовая комиссия из оффера.

## 9. Настройка метода QR-счёт

## 10. Настройка метода NSPK (link)

**Доступ:** admin

### Что такое NSPK

NSPK (Национальная система платёжных карт) — это метод оплаты через Систему быстрых платежей (СБП). Плательщик получает deeplink — ссылку, которая открывает банковское приложение для подтверждения платежа.

### Варианты методов

| Метод | Описание |
|-------|----------|
| **SBP** | Стандартный перевод через СБП по номеру телефона |
| **LOWSBP** | СБП с пониженными лимитами |
| **NSPK** | Перевод по deeplink-ссылке NSPK |

### Пошаговая настройка

1. **Создайте тип метода** «NSPK» или «СБП» (если ещё нет).

   - Тип метода: NSPK / СБП
   - Банки: привяжите банки, поддерживающие СБП

3. **Настройте оффер** с этим методом.

4. **Привяжите к трейдерам**, у которых есть реквизиты с номером телефона для СБП.

### Как это выглядит для плательщика

При оплате плательщик видит кнопку «Оплатить через СБП». При нажатии открывается его банковское приложение с предзаполненными данными перевода. После подтверждения средства поступают на реквизит трейдера.

## 12. Запуск трейдера в работу

### Чеклист запуска

Прежде чем трейдер начнёт получать заявки, необходимо выполнить все шаги:

- [ ] **Шаг 1.** Создать аккаунт трейдера (раздел 5)
- [ ] **Шаг 2.** Настроить Telegram-уведомления (указать ID чатов и токен бота)
- [ ] **Шаг 3.** Трейдер входит в личный кабинет и добавляет **реквизиты** (банковские карты / номера телефонов)
- [ ] **Шаг 4.** Подключить **устройство** для приёма SMS (раздел 20)
- [ ] **Шаг 5.** Привязать **офферы** к трейдеру и включить PayIn / PayOut (раздел 7)
- [ ] **Шаг 6.** Настроить **комиссии** по офферам
- [ ] **Шаг 7.** Пополнить **депозит** трейдера (раздел 13)
- [ ] **Шаг 8.** Включить **получение заявок** (переключатель на странице трейдера)

### Включение / выключение трафика

На странице трейдера вверху формы находится переключатель **Получение заявок**:
- **Включено** — трейдер получает новые заявки
- **Выключено** — новые заявки не поступают, но текущие можно завершить

Рядом отображается количество открытых споров.

![Скриншот: переключатель "Получение заявок" на странице трейдера](screenshots/trader-traffic-toggle.png)

### Условия получения заявок

Трейдер будет получать заявки только при соблюдении ВСЕХ условий:
1. Статус аккаунта = **Активный**
2. Переключатель **Получение заявок** = включен
3. Есть активные (не архивные) реквизиты для нужного метода/банка
4. У реквизита не исчерпан дневной лимит
5. Сумма заявки не превышает лимит на одну операцию
6. Достаточный депозит (баланс - страховой депозит - замороженные >= 0)
7. Хотя бы один оффер с нужным методом включён (PayIn или PayOut)
8. Не превышен лимит одновременных заявок

### Опция «Доверенные клиенты»

Если включена — трейдер получает заявки только от клиентов, уже совершавших платежи ранее. Новые клиенты (первый платёж) к нему не попадают. Снижает риск мошенничества.

## 14. Работа с апелляциями

### Как возникает апелляция

1. Плательщик совершает оплату, но заявка не подтверждается автоматически (проблема с SMS, таймаут и т.д.).
2. Заявка получает статус «Просрочен» или «Отменено».
3. Плательщик подаёт спор через форму на странице оплаты, прикладывая доказательства (скриншот чека).
4. Заявка переходит в статус **«Спор»**.

### Сроки

- Срок подачи апелляции: **14 дней** с момента создания заявки (по умолчанию, настраивается в системных настройках).

### Настройки на уровне трейдера

| Настройка | Описание |
|-----------|----------|
| **Право отмены апелляций** | Разрешает трейдеру самостоятельно отменять апелляции и изменять документы |
| **Лимит споров** | Максимальное количество споров. При превышении — автоблокировка трейдера |
| **Таймаут спора (мин)** | Время, после которого export-заявка в споре решается в пользу клиента |

### Уведомления

При создании спора уведомления отправляются:
- В **Telegram-чат споров** трейдера
- В чат саппорта (через Support Service)

### Статусы споров

| Статус | Описание |
|--------|----------|
| **Нет оплаты** | Трейдер отклонил спор — платёж не обнаружен |
| **Фейк чек** | Трейдер отклонил — предоставленный чек поддельный |
| **Не наши реквизиты** | Оплата была произведена на другие реквизиты |
| **Запросить пруф** | Трейдеру нужны дополнительные доказательства (видео, выписка) |

## 15. Настройка автоматики

### Автоподтверждение по SMS

Основной механизм автоматики: устройство трейдера (SMS Forwarder) пересылает SMS-уведомления от банка в систему. Система парсит сумму из SMS и автоматически подтверждает заявку с совпадающей суммой.

#### Что нужно для работы автоматики

1. **Устройство** трейдера подключено и онлайн (раздел 20)
2. **Банк** настроен с паттернами парсинга (раздел 6):
   - **Break-паттерны** — для фильтрации нерелевантных SMS (списания, покупки)
   - **Amount-паттерны** — для извлечения суммы
   - **Card End-паттерны** — для определения карты по последним 4 цифрам
3. **Имя отправителя SMS** в настройках банка совпадает с реальным отправителем SMS

#### Процент разброса для автоматики

Настраивается в оффере (поле **Процент разброса для автоматики**):
- При значении 2% и сумме заявки 1000 система подтвердит SMS на сумму от 980 до 1020
- Также блокирует создание новых заявок с суммами в этом диапазоне
- Рекомендуемое значение: 0–2%

### Режим метчинга

Метчинг — автоматическое связывание входящих платежей (payin) с выплатами (payout). Включается на уровне трейдера (чекбокс **Режим метчинга** в настройках трейдера).

Когда метчинг включён:
1. Трейдер в личном кабинете видит раздел **Метчинг**
2. Настраивает диапазоны сумм для связывания
3. Система автоматически подбирает пары payin-payout

В оффере есть настройка **Только внутренний метчинг** — если включена, payout подбирается только из того же оффера.

Основная коммуникация по спорам ведётся через Telegram-бот:

1. При создании спора бот отправляет уведомление в чат трейдера и чат саппорта.
2. Сообщение содержит кнопки действий:

## 19. Интеграция с агрегатором

### Общее описание

Trade Service работает в связке с Aggregator. Агрегатор — основной сервис, который принимает платежи от мерчантов и направляет их в Trade для обработки через трейдеров.

```
Мерчант -> Aggregator -> Trade -> Трейдер
                                    |
                              SMS / Telegram
                                    |
                              Подтверждение
                                    |
                         Trade -> Aggregator -> Мерчант
```

### Что нужно настроить

Для связки Trade и Aggregator необходимо в настройках обоих сервисов указать:

| Параметр | Где | Описание |
|----------|-----|----------|
| URL агрегатора | Trade | Адрес API агрегатора для отправки callback-ов |
| Токен Trade | Оба | Токен для аутентификации запросов от Aggregator к Trade |
| Токен Aggregator | Оба | Токен для аутентификации callback-ов от Trade к Aggregator |

Настройки задаются администратором сервера в конфигурационных файлах.

### Как проверить работу интеграции

1. Убедитесь, что в системных настройках указан корректный URL агрегатора.
2. Создайте тестовую заявку через агрегатор.
3. Проверьте, что заявка появилась в Trade Admin в разделе **Транзакции**.
4. После закрытия заявки проверьте, что статус обновился в агрегаторе.

> **Важно:** Настройка интеграции — задача системного администратора. Если интеграция не работает — обратитесь к DevOps / техлиду.

- **Личный кабинет трейдера:** Меню -> **Устройства**

### Что такое SMS Forwarder

SMS Forwarder — мобильное приложение, установленное на телефоне трейдера. Оно перехватывает входящие SMS от банков и пересылает их в систему для автоматического подтверждения заявок.

### Подключение устройства

1. Трейдер устанавливает приложение SMS Forwarder на свой телефон.
2. В личном кабинете переходит в **Устройства**.
3. Нажимает **Добавить устройство**.
4. Система генерирует **QR-код**.
5. Трейдер сканирует QR-код в приложении SMS Forwarder.
6. Устройство привязывается к аккаунту.

![Скриншот: страница устройств с QR-кодом для привязки](screenshots/device-qr-binding.png)

### Привязка устройства к банку

После добавления устройства необходимо указать, от какого банка оно будет принимать SMS:

1. В списке устройств нажмите **Редактировать** на нужном устройстве.
2. Привяжите банк (или несколько банков).
3. Сохраните.

### Мониторинг устройств в админке

В разделе **Устройства** (админка) отображается:

| Колонка | Описание |
|---------|----------|
| **Устройство** | Название / модель |
| **Трейдер** | К какому трейдеру привязано |
| **Статус** | Онлайн / офлайн |
| **Последняя активность** | Когда устройство было на связи |

> **Важно:** Если устройство офлайн — автоматика по SMS не работает. Заявки не будут подтверждаться автоматически. Проверяйте статус устройств регулярно.

## 21. Аналитика и ликвидность

### Конверсия по офферам

На странице трейдера в таблице офферов отображаются колонки:
- **Конверсия 1ч** — процент успешных заявок за последний час (формат: успешных/всего (процент))
- **Конверсия 24ч** — то же за последние 24 часа

Эти показатели помогают оценить эффективность трейдера по каждому офферу.

![Скриншот: конверсия по офферам на странице трейдера](screenshots/trader-conversion-stats.png)

### Курсы валют

**Доступ:** support и выше

Боковое меню -> **Курсы валют**.

Позволяет:
- Просматривать актуальные курсы из Rate Service (фильтр по валютам, банкам, источникам)
- Ручное обновление курса (кнопка **Добавить/обновить курс**)
- Сброс кэша конкретной комбинации курса
- Просмотр инструкции по работе с курсами (кнопка **Инструкция**)

### Транзакции и экспорт

**Доступ:** support и выше

Боковое меню -> **Транзакции**.

Здесь можно:
- Просматривать все заявки с фильтрацией по статусу, мерчанту, трейдеру, дате
- Видеть детали каждой заявки
- Экспортировать данные

В панели Agradmin доступны дополнительные инструменты:
- Расширенный экспорт транзакций
- Фильтрация по провайдерам и методам
- Статистика по мерчантам

### Выручка агентов

**Доступ:** agent, agentBounded

Боковое меню -> **Выручка**.

Показывает комиссионные доходы агента от трейдеров. Рядом — раздел **Выводы** для управления выводом комиссионных.

### Ликвидность

Ликвидность определяется наличием свободных реквизитов и депозитов у трейдеров. Для оценки ликвидности обращайте внимание на:

1. **Количество онлайн-трейдеров** — в списке трейдеров (фильтр по статусу «Активный» + получение заявок включено).
2. **Доступные реквизиты** — реквизиты с незаполненным лимитом.
3. **Конверсия** — высокая конверсия = хорошая ликвидность.
4. **Настройки провайдеров** (в Agradmin) — группы провайдеров с лимитами определяют распределение потока между провайдерами.

> **Совет:** Если конверсия падает — проверьте: достаточно ли онлайн-трейдеров, не исчерпаны ли лимиты реквизитов, работают ли устройства (SMS Forwarder).
