# Rate Service — Сервис курсов валют

> **Стек:** Python
> **Репо:** `rate/`
> **Последнее обновление:** 2026-04-15 (автосинк)

---

## 1. Обзор

Rate Service — микросервис для получения и предоставления курсов валют. Используется агрегатором для расчёта курсов обмена в транзакциях.

---

## 2. Источники курсов (`source_exchanges`)

Формат поля `source_exchanges` в конфигурации провайдеров:

| Формат | Описание |
|--------|----------|
| `<exchange>` | Обычный источник (напр. `binance`, `kaspi`) |
| `static-<number>` | Фиксированный курс, возвращает число напрямую |
| `binancekaspi130-<side>` | Binance/Kaspi со стороной сделки, 1.30x множитель. `<side>` = `buy` или `sell` |
| `rapira<N>` | Rapira с надбавкой. **Формат числа критичен** — см. ниже |

### Формат слага `rapira` — надбавка к курсу

> **Важно:** способ записи числа определяет размер надбавки.

| Слаг | Надбавка | Пример |
|------|----------|--------|
| `rapira0.5` | +0.5% | `rapira0.5` |
| `rapira05` | +5% | `rapira05` |
| `rapira5` | +5% | `rapira5` |
| `rapira10` | +10% | `rapira10` |

`rapira05` и `rapira5` идентичны — оба означают +5%.

Чтобы задать +0.5% — используйте только `rapira0.5` (с точкой).

### Формат `binancekaspi130-<side>` (добавлен 2026-04-14)
Позволяет задать сторону напрямую в строке источника:
```
binancekaspi130-buy   → source=binancekaspi130, side=buy
binancekaspi130-sell  → source=binancekaspi130, side=sell
```
Это позволяет не передавать `side` отдельным параметром.

## 3. Эндпоинты

### GET /rate/relevant — все доступные пары и слаги

Возвращает список всех актуальных валютных пар с курсами и доступными источниками (`source_exchanges` слагами).

```
GET /rate/relevant
Authorization: Bearer <token>
```

Используй этот эндпоинт чтобы узнать какие слаги (`binance101215`, `rapira0.5`, `bybit200k` и др.) доступны прямо сейчас — список динамический и меняется.

### GET /rate — получить курс

```
GET /rate?currency=RUB&second_currency=USDT&source_exchanges=rapira0.5&side=buy
```

| Параметр | Обязательный | Описание |
|----------|-------------|----------|
| `currency` | ✅ | Базовая валюта (напр. `RUB`) |
| `second_currency` | ✅ | Котируемая валюта (напр. `USDT`) |
| `side` | ✅ | `buy` или `sell` |
| `source_exchanges` | — | Слаг источника курса. Если не передан — используется дефолтный |
| `bank_or_payment_system` | — | Привязка к банку/шлюзу. `all_banks` если нет привязки |

### PATCH /rate — установить курс вручную

```
PATCH /rate?currency=RUB&second_currency=USDT&source_exchanges=static-91.5&side=buy&rate=91.5
```

Устанавливает курс в Redis (для обоих направлений `buy`/`sell`).
